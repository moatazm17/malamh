## 4. Web Scanning (Monitor Feature)

Make the Monitor's "Run scan" button do real web scanning instead of generating fake results.

**Database change:** Add a `source` column to `scan_results` table (string, default "demo", values: "google_lens" | "ai_platform" | "demo"). Also add `reference_image` column to the `faces` table (text, nullable) to store a base64-encoded 256x256 JPEG thumbnail.

**Save reference image:** During face registration, after IndexFaces succeeds, also save a compressed 256x256 JPEG version of the photo as base64 in the face's `reference_image` field.

**New endpoint:** GET /api/internal/face-image/:faceId — session auth, reads the face's `reference_image` base64, decodes it, responds with Content-Type: image/jpeg. This gives external APIs a URL to fetch the image.

**Scanning logic in POST /api/monitor/scan:**

1. Get the user's faces that have a `reference_image`
2. For each face:

**Source A — SerpAPI Google Lens** (if SERPAPI_KEY secret is set):
```
GET https://serpapi.com/search?engine=google_lens&url=<YOUR_APP_URL>/api/internal/face-image/<faceId>&api_key=<SERPAPI_KEY>
```
Parse `visual_matches` array from response. Each item has: `link` (page URL), `title`, `source` (domain), `thumbnail`.

**Source B — Lexica.art** (FREE, no key needed):
```
GET https://lexica.art/api/v1/search?q=portrait+face+photo
```
Returns array of AI-generated images with `src` URLs.

**Verification:** For each candidate from both sources (max 15 per source):
- Download the image (fetch the thumbnail URL or src)
- Send to AWS Rekognition SearchFacesByImage against our collection
- Only keep results where AWS confirms a match with the user's face (score > 70%)

**Store results:** Create scan_results rows with real sourceUrl, sourceDomain, pageTitle, matchScore (from AWS), and source ("google_lens" or "ai_platform").

**Fallback:** If no SERPAPI_KEY and AWS is not set up, keep demo mode (random fake results).

**Frontend updates to Monitor page:**
- Multi-step progress indicator during scan: "Searching Google..." → "Scanning AI platforms..." → "Verifying matches with AWS..." → "Complete"
- Source badge on each result: blue badge "Google" / purple badge "AI Platform" / gray "Demo"
- Keep existing takedown modal and status management as-is
